# Define front matter schema

Nail down the YAML front matter fields for tickets and comments.

## Tickets

Required:
- `status` — one of the configured statuses (default: open/in-progress/done)
- `created` — date (YYYY-MM-DD)

Optional:
- `tags` — list of strings
- `assignee` — string
- `priority` — TBD (string? number? enum?)
- `parent` — ticket number (for sub-tasks)

## Comments

Required:
- `created` — date (YYYY-MM-DD)

Optional:
- `author` — string

## Open questions

- Should `status` have a default so you can omit it for new tickets?
  (Probably yes — default to `open`)
- Priority scheme — keep it simple (low/medium/high) or leave it
  free-form?
- Do we need a `type` field (bug, feature, task) or is that what tags
  are for?
- Should `created` include time or just date? Just date is simpler but
  loses ordering within a day.
