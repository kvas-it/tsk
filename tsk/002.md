# Figure out the tooling story

tsk is a format — tools are implementations. But what operations actually
need tooling vs. just editing files by hand?

Candidates for tooling:
- `tsk new` — create next ticket (scan for max NNN, increment, open editor)
- `tsk list` — list tickets with status filtering
- `tsk status NNN <status>` — change ticket status
- `tsk comment NNN` — add a comment

Questions:
- Is a Rust CLI the right first tool, or would a simple shell script suffice
  for now?
- Should the CLI be opinionated (subcommands) or generic (just helps with
  file operations)?
- What about AI agents — do they need any tooling or can they just read/write
  files directly?

Don't build anything yet. This is about deciding what's worth building.
