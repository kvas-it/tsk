# Add integration tests for CLI

Run the `tsk` binary in a temp directory and assert on its output.
This covers things unit tests can't: tree formatting, parent chain
display in `tsk show`, orphan handling, filtered ancestor markers.

## Scope

- `tsk new` / `tsk new --parent N` — file creation, front matter
- `tsk list` — flat listing, tree view, status filters with ancestors
- `tsk show` — header, body, parent chain, sub-task listing
- `tsk status` — status change output

## Approach

Use Rust's `Command` to run the built binary against a temp tsk
directory with `tsk.yaml`. Create fixture tickets as files, then
assert on stdout. Could use `assert_cmd` crate or keep it
zero-dependency with plain `Command`.
