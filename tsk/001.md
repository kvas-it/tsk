# Write the tsk format specification

We need a standalone spec document that defines the tsk format precisely
enough that someone could implement a compatible tool without reading our
code.

Should cover:
- File naming conventions (NNN.md, NNN/MMM.md)
- YAML front matter schema (required vs optional fields, types)
- Title convention (first H1)
- tsk.yaml schema and defaults
- Directory layout
- How ticket numbering works

The spec should be minimal — define what's needed, leave room for tools
to add their own conventions on top.

Right now the format is described in CLAUDE.md and project.md. This ticket
is about pulling it into a proper spec document (probably `SPEC.md` or
similar at the repo root).
