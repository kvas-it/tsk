# CLI support for task nesting

Add support for the `parent` field in the tsk CLI so that task
hierarchies are visible and easy to create.

## Spec

Add `parent` as a documented optional field in SPEC.md. Value is
a ticket number (integer). Arbitrary nesting depth is allowed.

## CLI changes

### Parse `parent`

- Add `parent: Option<u32>` to the `Ticket` struct.
- Extract `parent:` from front matter in `parse_ticket`.

### `tsk new --parent N "title"`

- Accept `--parent N` flag in `tsk new`.
- Write `parent: N` into the front matter of the new ticket.
- Validate that the parent ticket exists.

### `tsk list` — tree view

- Show children indented under their parents (2 spaces per level).
- Orphan children (parent doesn't exist) show at top level.
- Filtering: when a status filter hides a parent but not its children,
  still show the parent (dimmed/marked) to preserve tree structure.
- Column alignment shifts with indentation (no fixed columns).

### `tsk show N` — display children

- When showing a ticket that has children, list them at the bottom
  (number, status, title — same as `tsk list` format).
- Show a warning when parent/child status looks inconsistent (e.g.
  "all children done but parent still open").
